# Documentação

## Estrutura

- `docs/reviews/`: documentos principais de cada ciclo de code review.
- `docs/reviews/artifacts/`: artefatos auxiliares por ciclo, como impacto, cobertura e revisões temáticas.
- `docs/quality/`: baseline e documentos de higiene técnica.
- `docs/testing/`: estratégia, guia e consolidação de testes E2E.
- `docs/architecture/`: documentação mais estável da aplicação e segurança.
- `docs/planning/`: backlog e spikes técnicos.

## Navegação Rápida

- Review atual: `docs/reviews/CODE_REVIEW_V14.md`
- Backlog do painel: `docs/planning/PANEL_IMPROVEMENTS_BACKLOG.md`
- Spike de Vite: `docs/planning/VITE_SPIKE_V4.md`
- Guia E2E: `docs/testing/E2E_GUIDE.md`
- Schema Firestore: `docs/architecture/FIRESTORE_SCHEMA.md`

## Convenção

- documentos `CODE_REVIEW_*` ficam em `docs/reviews/`
- artefatos específicos de um ciclo ficam em `docs/reviews/artifacts/vN/`
- documentos técnicos permanentes devem preferir `docs/architecture/`, `docs/testing/`, `docs/quality/` ou `docs/planning/`
